# Phase 1 前端 UI 设计方案（MiniMall 用户前台）

- 日期：2026-05-21
- 阶段：Phase 1 - Customer Frontend
- 性质：设计走向方案（非 TaskMaster 任务交付物，用于动手写页面前对齐 look-and-feel）
- 配套文档：`.taskmaster/docs/phase1-customer-frontend-prd.txt`、`docs/phase1-frontend-prereview.md`、`docs/frontend-integration.md`

> 边界声明：本方案严格限定在 PRD 范围内——仅用户前台，无管理员后台 / RBAC / 购物车 / 地址 / 优惠券 / 真实支付 / 退款；所有数据通过 api-gateway canonical `/api/**` 获取，不直连服务端口、不调用 `/internal/**`、不发送 `X-User-Id` / `X-Username`。

---

## 1. 整体视觉风格

定位：简洁现代的轻量电商前台，"克制的精致"——干净、有呼吸感、信息层级清晰，不堆营销视觉、不做花哨动画。

- 气质：白色卡片 + 浅灰背景的卡片化布局；适度圆角与轻阴影营造层次；大量留白让商品和价格成为视觉焦点。
- 基调：单一主色（科技蓝）贯穿交互元素（按钮、链接、高亮），中性灰承载结构，状态色仅用于标签 / 提示。
- 一致性：基于 Element Plus 默认设计语言做轻定制（覆盖主色与圆角），保证组件风格统一、开发高效，避免自造一套设计系统。
- 响应式：PC 优先，居中容器（最大宽度 1200px）；移动端做基础栅格降列即可，不追求完整移动商城体验。

## 2. 页面布局方案

全局骨架：固定顶部导航 `AppHeader` + 居中内容容器 + 浅灰页面背景。

```
┌────────────────────────────────────────────────────────┐
│ [MiniMall]   商品   我的订单            alice ▾  退出登录 │  ← AppHeader(sticky)
├────────────────────────────────────────────────────────┤
│                                                          │
│         ╭──────── 居中容器 max-width:1200px ────────╮     │  ← 浅灰背景 #f5f6f8
│         │            页面内容（白色卡片）            │     │
│         ╰────────────────────────────────────────╯     │
└────────────────────────────────────────────────────────┘
```

导航根据登录态切换：未登录显示「商品 / 登录 / 注册」，已登录显示「商品 / 我的订单 / 用户名 / 退出登录」，当前路由高亮。

各页面布局：

- 登录 / 注册：全屏浅灰，居中单卡片（宽 ~400px）。左可选品牌说明区，右表单（PC 双栏，移动单栏堆叠）。

```
        ╭─────────────────────────────╮
        │  MiniMall            登录    │
        │  轻量电商演示前台   ┌───────┐ │
        │                    │用户名 │ │
        │                    │密码   │ │
        │                    │[登录] │ │
        │                    └没账号?注册┘
        ╰─────────────────────────────╯
```

- 商品列表：顶部标题 / 筛选条 + 商品卡片网格（PC 4 列 / 平板 2-3 列 / 移动 1-2 列）+ 底部分页。

```
商品                         [状态: 全部 ▾]   ← 仅状态筛选（无搜索，后端不支持 keyword）
┌────┐ ┌────┐ ┌────┐ ┌────┐
│card│ │card│ │card│ │card│
└────┘ └────┘ └────┘ └────┘
┌────┐ ┌────┐ ...
              ◀ 1 2 3 ▶
```

- 商品详情：左右分栏。左 = 商品展示区（占位图 / 首字母色块），右 = 信息 + 库存框 + 数量选择 + 下单按钮。移动端上下堆叠。

```
┌──────────────┬───────────────────────────┐
│              │ 商品名         [上架中标签] │
│   商品展示    │ ¥199.90                    │
│   (占位区)    │ 描述文本……                  │
│              │ ┌库存框: 可用100 锁定0 在库┐ │
│              │ 数量 [- 1 +]   [立即下单]   │
└──────────────┴───────────────────────────┘
```

- 下单确认：单列窄卡片，订单摘要 + 总价高亮 + 确认 / 返回按钮。
- 我的订单：PC 用表格（订单号 / 商品摘要 / 金额 / 状态标签 / 创建时间 / 操作），移动端降级为卡片列表 + 分页。
- 订单详情：多张信息卡片（订单信息 / 商品信息 / 支付信息）+ 可选简化时间线 + 底部操作区。
- 模拟支付：居中卡片，订单号 + 大字金额 + 状态 + 模拟支付按钮 + 支付成功结果区 + 演示说明。
- 403 / 404：居中插画式空态 + 返回按钮。

| --- | --- | --- |

> 未知枚举值兜底：显示原始字符串 + 中性灰标签（PRD §10.4）。

字体

- 字体族：系统字体栈 `-apple-system, "Segoe UI", "PingFang SC", "Microsoft YaHei", sans-serif`（无需引入 web font，省加载、跨平台一致）。
- 字号阶梯：标题 24 / 20、卡片标题 16、正文 14、辅助 13、价格 22（加粗）。
- 字重：标题 600，正文 400，价格 700。

间距（8px 基准栅格）

- 间距阶梯：4 / 8 / 12 / 16 / 24 / 32。
- 卡片内边距 24，卡片圆角 12，按钮 / 输入圆角 8。
- 阴影：`0 4px 16px rgba(0,0,0,.06)`（脚手架 `App.vue` 已用此值，全站统一）。
- 网格间距：商品卡片 gap 16~20。

## 4. 组件拆分

对齐 PRD §10.2，按"展示型 / 状态型 / 容器型"分层：

布局 / 容器
- `AppHeader` — 顶部导航，读 auth store 切换登录态、当前路由高亮、退出登录。
- `PageContainer`（可选）— 居中容器 + 标题槽，统一页面外边距。

展示型（纯 props，无副作用）
- `ProductCard` — 商品名 / 描述摘要 / 价格 / 状态标签 / 查看详情。
- `PriceText` — 统一金额格式（`¥` + 两位小数，处理 BigDecimal 数字）。
- `StatusTag` — 通用状态标签（枚举→中文文案 + 色），商品 / 库存 / 支付复用。
- `OrderStatusTag` — 订单状态专用（5 态配色）。
- `PaymentResultCard` — 支付成功结果展示。
- `OrderInfoCard` / `ProductInfoCard`（详情页信息卡）。

状态型（页面级通用占位）
- `LoadingState` — 骨架屏 / 加载态。
- `EmptyState` — 空数据插画 + 文案 + 行动按钮。
- `ApiErrorAlert` — 错误提示 + 重试按钮。

交互型
- `PaginationBar` — 分页（封装 page / size，对接 PageResponse）。
- `QuantityStepper` — 数量选择器（min 1，上限 = 库存）。
- `ConfirmDialog` — 二次确认（取消订单等）。

> 状态型 / 交互型组件优先封装，因为每个 API 页面都要复用（PRD §10.3 要求每页都考虑 loading / empty / error）。展示型可按需，避免过度抽象。

## 5. 每个页面的关键交互状态

| 页面 | 关键交互状态 |
| --- | --- |
| 登录 | 表单校验（用户名 / 密码非空）；按钮 loading；成功跳 redirect 或 `/products`；失败展示后端 message；401 不崩溃；入口"去注册" |
| 注册 | 校验（用户名非空、密码 ≥6 位对齐后端、email 非空时合法、phone ≤32）；按钮 loading；成功提示后跳 `/login`；失败展示 message |
| 商品列表 | loading（骨架）/ empty（无商品）/ error（重试）；状态筛选切换；下架商品弱化（灰标 + 弱化入口）；分页切换；点击进详情 |
| 商品详情 | 商品 + 库存并行加载；商品不存在→error；下架禁用下单；库存不足 / OUT_OF_STOCK / INACTIVE 禁用下单；数量受库存上限约束；未登录点下单→跳 `/login?redirect=` |
| 下单确认 | 进页生成 idempotencyKey；商品 / 库存复查；下架或缺货禁止确认；按钮 loading 防重复点击；成功跳 `/orders/{orderNo}`；失败恢复按钮 |
| 我的订单 | 未登录→跳登录；loading / empty / error；按状态显示操作（待支付→去支付 / 取消；其它→查看详情）；取消需 ConfirmDialog，成功刷新列表；分页 |
| 订单详情 | 未登录→跳登录；不存在 / 无权→404 文案（后端越权返 404）；按状态显操作（PENDING_PAYMENT→去支付 / 取消；PAID→查看支付信息）；取消 / 支付后重拉详情 |
| 模拟支付 | 仅 PENDING_PAYMENT 显示支付按钮；点击 loading；成功后展示成功态并短轮询订单状态（3~5 次，500~1000ms）确认 PAID；超轮询提示"已提交，稍后刷新"（不判失败）；明确"模拟支付"说明 |

## 6. loading / empty / error / success 设计

统一四态语言，全站一致：

- Loading
  - 列表 / 卡片：骨架屏（灰色占位块，优于转圈，减少跳动感）。
  - 按钮 / 提交：按钮内联 spinner + 禁用（防重复提交）。
  - 详情页：分区骨架（商品区、信息区各自骨架）。
- Empty
  - 居中淡色线性图标 + 主文案 + 副文案 + 行动按钮。例：我的订单空 → "还没有订单 / 去逛逛商品" → 按钮跳 `/products`。
- Error
  - `ApiErrorAlert`：浅红底警告条 + 后端 message（或友好兜底）+「重试」按钮；不暴露堆栈、不崩页。
  - 全局类错误由 API client 统一 toast：401「登录已过期」+ 跳登录、429「请求过于频繁，请稍后再试」、500「系统异常」、网络「网络连接异常，请稍后重试」。
- Success
  - 轻量 toast（注册成功、取消成功）。
  - 支付成功用 `PaymentResultCard`：绿色对勾 + "支付成功" + 金额 / 支付单号 + "查看订单"按钮——给闭环一个明确的正反馈终点。

## 7. 用户购买闭环页面流转

```
                         ┌─────────────┐
  游客 ── 浏览 ──────────▶│ /products   │ 商品列表
                         └──────┬──────┘
                                │ 点击商品
                                ▼
                         ┌─────────────┐
                         │/products/:id│ 商品详情(+库存)
                         └──────┬──────┘
                       点击"立即下单"
                                │
                未登录 ─────────┼───────── 已登录
                   │            │
                   ▼            ▼
           /login?redirect ─登录成功→ ┌──────────────┐
                                     │ /checkout    │ 下单确认(idempotencyKey)
                                     └──────┬───────┘
                                      创建订单成功
                                            ▼
                                     ┌──────────────────┐
                                     │/orders/:orderNo  │ 订单详情(PENDING_PAYMENT)
                                     └──────┬───────────┘
                                       点击"去支付"
                                            ▼
                                     ┌──────────────────┐
                                     │/payments/:orderNo│ 模拟支付
                                     └──────┬───────────┘
                                   支付成功→短轮询订单
                                            ▼
                                  订单状态 PAID ✔（成功结果卡）
                                            │
                                   "查看订单" / "我的订单"
                                            ▼
                                     ┌──────────────┐
                                     │ /orders      │ 我的订单列表
                                     └──────────────┘
```

旁路：详情页 / 订单页对待支付订单提供"取消"（ConfirmDialog → 取消成功刷新）。`/` 重定向 `/products`；受保护路由未登录统一 `/login?redirect=`。

## 8. 提升美感但不扩大业务范围的设计细节

这些只动"观感 / 微交互"，不新增任何业务功能、不新增接口依赖：

1. 商品占位图：无图商品用"品牌色块 + 商品名首字"生成稳定占位（按 productId 哈希取色），避免大片空白。
2. 骨架屏替代转圈，减少加载跳动，质感更高。
3. 状态标签统一为彩色 Tag（圆角、淡底色 + 深字），不显示原始枚举串。
4. 价格排版：`¥` 小、整数大、小数小的混排，价格更醒目。
5. 卡片 hover 微交互：商品卡轻微上浮 + 阴影加深（≤150ms），克制不浮夸。
6. 支付成功的对勾动画（一次性、短）：给闭环正反馈，纯展示。
7. 空状态线性插画 / 图标：统一风格，提升空页面亲和力。
8. 简化订单时间线（创建→待支付→支付成功）：纯展示，数据全部来自现有订单字段（createdAt / paidAt 等），不引入新接口。
9. 页面进入轻微淡入（透明度过渡），避免生硬。
10. 数量选择器、按钮 disabled 态的清晰视觉：让"为什么不能下单"（下架 / 缺货）一眼可辨。
11. 金额 / 库存等数字用等宽或 tabular-nums，对齐美观。
12. 顶部导航 sticky + 细微底部阴影，滚动时层次清晰。
